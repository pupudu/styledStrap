# Card
<Helpers.Subtitle>Flexible and extensible content container with multiple variants and options</Helpers.Subtitle>

## Example
Below is an example of a basic card with mixed content and a fixed width.

Cards have no fixed width to start, so they’ll naturally fill the full width of its parent element
unless specified explicitly like in this example.

<Preview>
  <BasicCard />
</Preview>

```jsx
<Card width="18rem">
  <CardImg src={src} alt="swiss-landscape" />
  <CardBody>
    <Heading size="h5">Card Title</Heading>
    <p>
      Some quick example text to build on the card title and make up the bulk of the card's
      content.
    </p>
    <Button>Go Somewhere</Button>
  </CardBody>
</Card>
```

Cards support a wide variety of content, including images, text, list groups, links, and more.

Create lists of content in a card with a flush list group. The flush prop is
passed to ignore the default left and right borders of the ListGroup.

### Header and Footer
Add an optional **header** and/or **footer** within a card.

<Preview>
  <OptionalHeader />
</Preview>

```jsx
<Card>
  <CardHeader>Optional Header</CardHeader>
  <CardBody>
    <CardTitle>Special title treatment</CardTitle>
    <CardText>With supporting text below as a natural lead-in to additional content.</CardText>
    <Button>Go Somewhere</Button>
  </CardBody>
  <CardFooter>Optional Footer</CardFooter>
</Card>
```

You can use the StyledComponents' `as` prop to change the Header component to use
any tag or component you like. Below you see how we can change the tag to an "h5".
Similarly, you can change it to use the stylestrap `Heading` component and make
use of all its props.

<Preview>
  <OptionalHeaderBold />
</Preview>

```jsx
<Card>

  <CardBody>
    <CardTitle>Special title treatment</CardTitle>
    <CardText>With supporting text below as a natural lead-in to additional content.</CardText>
    <Button>Go Somewhere</Button>
  </CardBody>
</Card>
```

## Sizing
Cards assume no specific width to start, so they’ll be 100% wide unless otherwise stated.
You can change this as needed either using the `width` prop, or by embedding the cards
within a grid layout.

<Preview>
  <Card width="36rem">
    <CardBody>
      <CardTitle>Using Width Prop</CardTitle>
      <CardText>
        The width of this card is explicitly set using the 'width' prop.
        Width prop accepts values in all supported units of CSS (px, rem, em, %).
      </CardText>
      <Button>Button</Button>
    </CardBody>
  </Card>
</Preview>

```jsx
<Card width="36rem">
  <CardBody>
    <CardTitle>Using Width Prop</CardTitle>
    <CardText>
      The width of this card is explicitly set using the 'width' prop.
      Width prop accepts values in all supported units of CSS (px, rem, em, %).
    </CardText>
    <Button>Button</Button>
  </CardBody>
</Card>
```

The same effect can be achieved by making the Card a child of a Grid.
The Grid's column prop can be changed to give the card either a fixed width or a relative width.
Similarly 'Row' and `Column` components can be used to control the Card size
according to the Flexbox layout.

```jsx
<Grid columns="36rem 1fr">
  <Card>
    {/* Card content */}
  </Card>
</Grid>
```

## Navigation
Add some navigation to a card’s header with the CardHeaderNav component.

<Preview>
  <Card>
    <CardHeader>
      <CardHeaderNav tabs>
        <NavLink href="#" active>Active</NavLink>
        <NavLink href="#">Other</NavLink>
        <NavLink href="#" disabled>Disabled Tab</NavLink>
      </CardHeaderNav>
    </CardHeader>
    <CardBody>
      <CardTitle>Special title treatment</CardTitle>
      <CardText>With supporting text below as a natural lead-in to additional content.</CardText>
      <Button>Go Somewhere</Button>
    </CardBody>
  </Card>
</Preview>

```jsx
<Card>
  {/* Header with tabs */}
  <CardHeader>
    <CardHeaderNav tabs>
      <NavLink href="#" active>Active</NavLink>
      <NavLink href="#">Other</NavLink>
      <NavLink href="#" disabled>Disabled Tab</NavLink>
    </CardHeaderNav>
  </CardHeader>
  {/* Followed by a regular card body */}
  <CardBody>
    <CardTitle>Special title treatment</CardTitle>
    <CardText>With supporting text below as a natural lead-in to additional content.</CardText>
    <Button>Go Somewhere</Button>
  </CardBody>
</Card>
```

## Images
Cards include a few options for working with images.
Choose from appending “image caps” at either end of a card, overlaying images with card content,
or simply embedding the image in a card.

### Image Caps
Similar to headers and footers, cards can include images at the top, bottom or middle of a card.

<Preview>
  <CardImagePreview />
</Preview>

```jsx
import src from './cardImg.jpg';
//...
<Grid columns="3">
  <Card width="15rem">
    <CardImg src={src} />
    <CardBody>
      <CardText>Some text description of the image below the image</CardText>
    </CardBody>
  </Card>

  <Card width="15rem">
    <CardBody>
      <CardText>Some text description of the image above the image</CardText>
    </CardBody>
    <CardImg src={src} />
  </Card>

  <Card width="15rem">
    <CardBody>
      <CardText>Some text above the image</CardText>
    </CardBody>
    <CardImg src={src} />
    <CardBody>
      <CardText>Some text below the image</CardText>
    </CardBody>
  </Card>
</Grid>
```

### Image Overlays
Turn an image into a card background and overlay your card’s text.

<Preview>
  <CardImageOverlay />
</Preview>

```jsx
<Card width="20rem" textColor="#FFF">
  <CardImg src={src} />
  <CardImgOverlay>
    <CardTitle>Card title</CardTitle>
    <CardText>
      This is a card with supporting text below as a natural lead-in to additional content. This
      content is a little bit longer.
    </CardText>
    <CardText>Last updated 3 minutes ago</CardText>
  </CardImgOverlay>
</Card>
```

<WarnCard>
Note that content should not be larger than the height of the image.
If content is larger than the image the content will be displayed outside the image.
<b> See the <a href="#image_size">Image Size</a> section for an alternative way to do this.</b>
</WarnCard>

### Horizontal
Cards with images can be made horizontal in a mobile-friendly and responsive way,
easily by using one of Grid or Flex layouts inside the card.

In the following example, we use a 2 column Grid inside the card.

<Preview>
  <CardImageHorizontal />
</Preview>

```jsx
<Card width="40rem">
  <Grid columns="1fr 1fr">
    <CardImg src={src} />
    <CardBody>
      <CardTitle>Card title</CardTitle>
      <CardText>
        This is a card with supporting text below as a natural lead-in to additional content.
        This content is a little bit longer.
      </CardText>
      <CardText muted>Last updated 3 minutes ago</CardText>
    </CardBody>
  </Grid>
</Card>
```

### Image Size
The width of the `CardImg` is set to 100% by default. This can make it a bit tricky
to correctly use an image either as an overlay, or as a horizontal image inside a card.
Thus unlike in raw bootstrap, stylestrap exports a BgImg component which allows
us to control the background size of the image using css properties.

From the two cards below, the first card uses the BgImg component, and use the
background size of "cover"(default). More details about this property can be found
in [MDN Docs](https://developer.mozilla.org/en-US/docs/Web/CSS/background-size)
<Preview>
  <CardBgImg />
</Preview>

```jsx
<Card width="25rem">
  <Grid columns="2">
    <BgImg src={src} />
    <CardBody>
      <CardTitle>Card title</CardTitle>
      <CardText>
        Image fits perfectly on the space available, by cropping the image horizontally.
      </CardText>
      <CardText muted>Last updated 3 minutes ago</CardText>
    </CardBody>
  </Grid>
</Card>
```
Similarly, the issue with Image Overlays can be solved by
wrapping CardBody in a BgImg component.

<Preview>
  <CardBgImgOverlay />
</Preview>

```jsx
<Card width="18rem" textColor="white">
  <BgImg src={src}>
    <CardBody>
      <CardTitle>Card title</CardTitle>
      <CardText>
        This content is a little bit longer. This image is cropped to fit both the content
        and image without any unintended white spaces
      </CardText>
      <CardText>Last updated 3 minutes ago</CardText>
    </CardBody>
  </BgImg>
</Card>
```

## Card Styles

Stylestrap cards come in different flavors for customizing their backgrounds, borders, and color.
You can use preset styles by passing the `color` and `flavor` props, and also override the default
preset styles using other props.

### Plain Background Color
Set the main background color of the card by setting the color prop.

<Preview>
  <CardStylePlain />
</Preview>

```jsx
<Card color="primary">
  <CardHeader>Heading</CardHeader>
  <CardBody>
    <CardTitle>Card Title</CardTitle>
    <CardText>
      Some quick example text to build on the card title and make up the bulk of the card's
      content.
    </CardText>
  </CardBody>
</Card>

<Card color="secondary">{/* Similar content as above */}</Card>
<Card color="success">{/* Similar content as above */}</Card>
<Card color="warning">{/* Similar content as above */}</Card>
<Card color="danger">{/* Similar content as above */}</Card>
<Card color="info">{/* Similar content as above */}</Card>
<Card color="light">{/* Similar content as above */}</Card>
<Card color="dark">{/* Similar content as above */}</Card>
```

Notice that the text color is derived based on the luminosity of the
main color. You can take control of this by passing the `textColor` prop explicitly.

<Preview>
  <PlainCard color="dark" textColor="warning" width="18rem" />
</Preview>

```jsx
<Card color="dark" textColor="warning" width="18rem">
  {/* Similar content as above */}
</Card>
```
