## Examples
Stylestrap includes several predefined button styles, each serving its own semantic purpose.

Note that unlike in raw bootstrap, we don't include the "link" style in our buttons.
Instead, we have dedicated Link and Anchor components, which are more flexible
and easier to extend.

## Button tags
The buttons by default uses the html button tag. However, just like any other stylestrap component,
can be customized to use other tags like `input` or `a` using the styled-components' `as` prop.

When you set the tag as `a`(anchor), stylestrap will automatically
set the `role` attribute as "button".

<Preview>
  <Spaced>
    <Button as="a">Primary</Button>
    <Button as="input" type="button" value="Input" />
    <Button as="input" type="submit" value="Submit" />
    <Button as="input" type="reset" value="Reset" />
  </Spaced>
</Preview>

```jsx
<Button as="a">Primary</Button>
<Button as="input" type="button" value="Input" />
<Button as="input" type="submit" value="Submit" />
<Button as="input" type="reset" value="Reset" />
```

## Loading Buttons
In a real world web application, data fetching on response to a button click is a common scenario.
Thus stylestrap includes a Loading Button preset which can provide visual indications to show the
progress.

In the following example, the loading indicator is added based on a timer.
In a real world application, this would be replaced by a fetch call or something similar.

<Preview>
  <Presets.LoadingButton>LoadingButton</Presets.LoadingButton>
</Preview>

```jsx
function LoadingButtonExample() {
  const [loading, setLoading] = useState(false);
  return (
    <LoadingButton
      loading={loading}
      onClick={() => {
        setLoading(true);
        setTimeout(() => setLoading(false), 2000);
      }}
    />
  );
}
```
